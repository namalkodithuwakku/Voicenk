# Development Rules

1. Build one feature at a time.
2. Test before starting the next feature.
3. Keep feature logic inside its own feature folder.
4. Avoid large multi-purpose components.
5. Do not expose secret keys to the client.
6. Use full replacement files when handing changes to the project owner.
7. Avoid unrelated changes.
8. Preserve working production behavior.
9. Commit each tested milestone separately.
10. ZIP packages are used for starters, releases and backups.
